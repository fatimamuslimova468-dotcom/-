1. Введение

«Код Безопасности» – российский сервис для комплексного аудита безопасности мессенджеров. Мы проверяем уязвимости, шифрование, соответствие законодательству (152‑ФЗ, GDPR) и функциональную стабильность. После проверки вы получаете официальное заключение в формате PDF с усиленной ЭЦП.

Сервис работает по модели «сначала оплата, потом анкета». Все заявки сохраняются в Firebase, файлы исходного кода загружаются в облачное хранилище.

3. Оплата через Lava

При нажатии «Оплатить через Lava» генерируется прямая ссылка с вашим Profile ID и суммой.

https://lava.ru/pay/1f20c4a6-ff03-4628-86c2-7dab1ad09db5?amount=200&comment=Проверка+безопасности&return_url=https://codbez.zapto.org/success.html

Параметры:

  • shop_id – ваш публичный идентификатор магазина
  • amount – сумма в рублях
  • comment – описание платежа
  • return_url – страница возврата после оплаты (у нас /success.html)

После успешной оплаты Lava перенаправляет пользователя на success.html, где статус проверяется через API Lava. При положительном ответе в localStorage устанавливается флаг payment_verified=true, и после возврата на главную страницу автоматически открывается анкета.

4. Аутентификация пользователей

На сайте реализована регистрация и вход через Firebase Auth. Поддерживаются:

  • Email/пароль
  • Вход через Google (OAuth)

После входа в шапке отображается аватар и имя пользователя. Все заявки привязываются к UID пользователя (если он авторизован).

5. Работа с Firebase

Основные коллекции Firestore:

  • consultations – заявки на консультацию
  • audit_orders – заказы на аудит

6. Анкета аудита

После подтверждения оплаты открывается модальное окно с полями:

  • Ваше имя / название организации
  • Файл с исходным кодом (zip, rar, 7z)
  • Email или способ для ответа

После отправки данные сохраняются в Firestore, файл загружается в Storage.

7. API и Webhook

Пример обработчика Webhook на Node.js:

const crypto = require('crypto');
app.post('/lava-webhook', (req, res) => {
  const signature = req.headers['x-lava-signature'];
  const hash = crypto.createHmac('sha256', 'доп.ключ')
                     .update(JSON.stringify(req.body))
                     .digest('hex');
  if (hash === signature) {
    // платёж подтверждён
  }
  res.sendStatus(200);
});

8. Часто задаваемые вопросы

В: Что делать, если после оплаты не открывается анкета?
О: Проверьте, что вы попали на страницу success.html, и в localStorage есть ключ payment_verified.

В: Можно ли заказать аудит без регистрации?
О: Да, регистрация необязательна.

В: Как долго выполняется проверка?
О: Стандартный срок — от 3 до 7 рабочих дней.
